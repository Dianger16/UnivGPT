# UniGPT Demo Run Log

> Simulated end-to-end demo showing: Admin creates account → uploads student doc → student logs in → queries → receives answer with citations.

```

```

{

  "token_type": "bearer",
  "user": {
    "id": "00000000-0000-0000-0000-000000000001",
    "email": "admin@university.edu",

## Step 3: Admin Uploads a Student Document

```
POST http://localhost:8000/admin/documents
Authorization: Bearer <admin_token>
Content-Type: multipart/form-data

file: student_handbook_2026.pdf
doc_type: student
department: General
course: ALL
tags: ["handbook", "policies", "student-life"]

Response 200:
{
  "id": "10000000-0000-0000-0000-000000000001",
  "filename": "student_handbook_2026.pdf",
  "doc_type": "student",
  "department": "General",
  "course": "ALL",
  "tags": ["handbook", "policies", "student-life"],
  "visibility": true
}

# Behind the scenes:
# - File stored in Supabase Storage at documents/student/...
# - Text extracted using pdfplumber
# - Text chunked into 3 chunks (1000 chars, 200 overlap)
# - Embeddings generated via OpenRouter (or mock mode)
# - 3 rows inserted into document_texts
# - 3 rows inserted into embeddings with metadata.role = "student"
# - Audit log created: action = "document_upload"
```

## Step 6: Student Tries to Access Sensitive Data — Blocked

{
  "query": "What are my grades this semester?"
}
